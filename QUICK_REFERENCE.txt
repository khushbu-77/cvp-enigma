ENDPOINTS

  POST /issue         → Process certificate issuance
  GET  /verify        → Verify Certificate form
  POST /verify/result → Check certificate validity

  ├─ certificate_id (Primary Key)

  ├─ father_name ← NEW
  ├─ dob ← NEW
  ├─ course

CERTIFICATE FLOW

  ISSUE PHASE:
  1. User fills form with student details
  2. System generates unique ID (CERT-XXXXXXXX)
  3. Data normalized and hashed (SHA-256)
  4. Certificate saved to database
  5. User shown certificate ID

  VERIFY PHASE:
  1. User enters certificate ID
  2. Database lookup for certificate
  3. Data re-normalized and re-hashed
  4. Hash compared with stored hash
  5. Status displayed: VALID / INVALID / TAMPERED

TROUBLESHOOTING

  Q: Port 5000 in use?
  A: python app.py --port 5001

  Q: Database corrupt?
  A: Delete database.db and restart

  Q: Import errors?
  A: Ensure you're in cvp-enigma directory

  Q: Tests failing?
  A: Run: python test_app.py (for debugging output)
